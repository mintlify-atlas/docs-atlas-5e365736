This guide will walk you through training your first custom Soprano model using the provided example dataset. You'll go from installation to a trained model ready for inference.

## Prerequisites

Before you begin, ensure you have:

- Python 3.8 or later installed
- CUDA-compatible GPU (recommended for faster training)
- At least 8GB of GPU memory for training

<Note>
If you're on Windows, you may need to reinstall PyTorch with CUDA support after installing the requirements.
</Note>

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone Soprano-Factory from GitHub and navigate to the project directory:

    ```bash
    git clone https://github.com/ekwek1/soprano-factory.git
    cd soprano-factory
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required packages using pip:

    ```bash
    pip install -r requirements.txt
    ```

    The requirements include:

    ```txt
    einops
    huggingface_hub
    numpy
    scipy
    torch
    torchaudio
    tqdm
    transformers
    ```

    <Info>
    On Windows, you may need to reinstall PyTorch to enable CUDA support:
    ```bash
    pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118
    ```
    </Info>
  </Step>
</Steps>

## Prepare your dataset

Soprano-Factory expects datasets in **LJSpeech format**. An example dataset is provided in the `example_dataset` directory.

### Dataset structure

Your dataset should follow this structure:

```plaintext
example_dataset/
├── metadata.txt
└── wavs/
    ├── example1.wav
    └── example2.wav
```

### Metadata format

The `metadata.txt` file contains one line per audio file with the format `filename|transcript`:

```txt
example1|Soprano is an extremely lightweight text to speech model designed to produce highly realistic speech at unprecedented speed.
example2|Gabagool? Ova here!
```

<Note>
WAV files can be in any sample rate - they will be automatically resampled to 32 kHz during preprocessing.
</Note>

## Train your model

<Steps>
  <Step title="Preprocess your dataset">
    Convert your audio files into tokens that Soprano can train on:

    ```bash
    python generate_dataset.py --input-dir example_dataset
    ```

    This script will:
    - Load the Soprano encoder from HuggingFace
    - Encode all audio files to tokens
    - Create train/val splits (90/10 ratio, max 512 validation samples)
    - Save `train.json` and `val.json` in your dataset directory

    <Info>
    If you omit the `--input-dir` argument, it defaults to `./example_dataset`.
    </Info>
  </Step>

  <Step title="Start training">
    Begin training your custom Soprano model:

    ```bash
    python train.py --input-dir example_dataset --save-dir ./my_model
    ```

    The training script will:
    - Load the base Soprano-80M model from HuggingFace
    - Train for 10,000 steps with validation every 250 steps
    - Use AdamW optimizer with warmup and cooldown scheduling
    - Save the final model to your specified directory

    ### Training hyperparameters

    The default configuration in `train.py` includes:

    ```python
    max_lr = 5e-4
    batch_size = 4
    seq_len = 1024
    max_steps = 10000
    val_freq = 250
    ```

    <Info>
    Training runs on `cuda:0` by default. The script uses mixed precision (bfloat16) for efficient memory usage.
    </Info>
  </Step>

  <Step title="Monitor training progress">
    During training, you'll see progress updates showing:

    ```
    text loss: 0.123 | audio loss: 0.456 | acc: 0.8234 | lr: 5.00e-04 | norm: 0.123 | time: 45.67 ms | 9123.45 t/s
    ```

    Validation metrics are printed every 250 steps:

    ```
    validation text loss: 0.125 | validation audio loss: 0.458 | validation acc: 0.8201
    ```
  </Step>
</Steps>

## Use your trained model

Once training is complete, you can use your custom model with the main [Soprano repository](https://github.com/ekwek1/soprano).

```python
from soprano import Soprano

# Load your custom model
tts = Soprano(model_path="./my_model")

# Generate speech
audio = tts.generate("Hello from my custom Soprano model!")
```

<Note>
Your trained model is fully compatible with all Soprano inference features including WebUI, CLI, and OpenAI-compatible endpoints.
</Note>
